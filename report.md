# 用户画像-投资策略匹配：数据分析报告

**版本**：V2.0
**日期**：2026-05-23
**方法**：基于 PCA + 径向惩罚余弦相似度的风格偏好匹配

---

## 一、项目背景与核心问题

在金融投资领域，"什么样的策略推荐给什么样的客户"是一个经典问题。本项目需要解决的问题可以表述为：

> 给定一组量化投资策略的历史交易数据，以及一组客户的交易记录，如何为每个客户推荐最匹配的投资策略？

我们面临的约束条件：
- **策略侧**：拥有完整的净值曲线 + 交易明细 + 资金流水（7 个策略，约 3 年历史）
- **用户侧**：仅有交易流水记录（时间、标的、买卖方向、价格、数量），**无期初资金、期末资产、现金余额**
- **数据量**：3 个模拟用户（A: 127 笔, B: 1727 笔, C: 285 笔交易）

---

## 二、核心假设体系

整个匹配方案建立在以下四个基本假设之上：

### 假设 1：客户都期望在可接受的风险范围内获得更高收益

**含义**：没有客户会主动追求低收益。因此，收益率不作为"匹配维度"，而是作为**硬性过滤条件**。

**落地方式**：
- 策略的年化收益 ≥ 用户历史收益 → 否则不推荐
- 策略的最大回撤 ≤ 用户可承受回撤 → 否则不推荐

**金融直觉**：如果一个策略的年化收益是 5%、用户过去自己做的年化是 3%，用户不会因为这个"接近"就接受推荐 — 用户期望的是更高的收益。

### 假设 2：客户对风格一致的投资策略信任度更高

**含义**：用户更愿意接受一个"看起来像我自己会选的策略"。风格一致性（持仓周期、换手率、行业集中度等行为特征的对齐）比绝对收益率的相似更重要。

**金融直觉**：一个习惯长线持有的用户看到频繁调仓的短线策略，会本能地不信任；一个习惯集中持仓的用户看到高度分散的策略，会觉得"不像我的风格"。风格一致意味着用户可以用自身的先验经验来理解和评估推荐策略。

### 假设 3：无法获得期初资金时，应从"计算绝对收益率"转向"刻画行为画像"

**含义**：用户的交易流水不包含账户总资产信息，强行估算绝对收益率会引入巨大噪声（因为初始资金、底仓结构等关键信息缺失）。应改为提取稳健的行为特征来刻画用户画像。

**金融直觉**：即使不知道用户赚了多少，知道他"喜欢做什么"已经足够匹配到同类型的策略。在用户画像领域，行为特征往往比收益特征更稳定、更可解释。

### 假设 4：用户与策略可以在同一套特征体系下被度量

**含义**：从策略的交易行为中提取的特征，也可以从用户的交易行为中以相同方式提取。两者因此可以映射到同一个特征空间。

**金融直觉**：无论是策略还是用户，"持仓周期"、"换手率"、"买卖对称性"等概念的定义是普适的。如果策略 A 的平均持仓周期是 12 天，用户 B 的平均持仓周期是 52 天，这可以直接比较。

## 三、三层特征体系

基于上述假设，我们构建了 **12 维特征体系**，分为三个层次：

### 第一层：交易行为特征（6 维）

| 特征 | 计算方法 | 刻画含义 | 数据需求 |
|------|---------|---------|---------|
| 持仓周期 (holding_period) | FIFO 配对后，买入→卖出的天数中位数 | 长线型 vs 短线型 | 交易流水 |
| 换手率 (turnover_rate) | 总交易笔数 / 自然天数 | 活跃度 vs 静默型 | 交易流水 |
| 买卖对称性 (buy_sell_ratio) | 买入金额 / 卖出金额 | 建仓型 vs 波段型 | 交易流水 |
| 持仓集中度 (hhi_concentration) | 各标的交易金额占比的 HHI 指数 | 集中 vs 分散 | 交易流水 |
| 处置效应系数 (disposition_effect) | 已实现盈利单数占比 / 已实现亏损单数占比 | 止损意识强弱 | 交易流水 |
| 正收益占比 (positive_trade_ratio) | 闭环交易中盈利笔数 / 总闭环数 | 胜率偏好 | 交易流水 |

**关键性质**：这 6 个特征完全从交易流水中计算，不需要任何资金、净值、行情数据，**零外部依赖**。

### 第二层：资产偏好特征（3 维）

| 特征 | 计算方法 | 刻画含义 |
|------|---------|---------|
| ETF 偏好 (etf_ratio) | ETF 标的交易笔数占比 | 偏好被动配置 vs 主动选股 |
| 价格区间偏好 (avg_price_preference) | 买入均价 | 低价股偏好 vs 高价股偏好 |
| 分仓均匀度 (position_uniformity) | 1/(1+CV)，单笔金额变异系数的倒数 | 均匀分仓 vs 重仓集中 |

**关键性质**：不涉及收益率，只刻画"用户喜欢买什么"，用于与策略的底层资产类型做匹配。

### 第三层：风险代理特征（3 维）

| 特征 | 计算方法 | 刻画含义 |
|------|---------|---------|
| 亏损幅度 (avg_loss_magnitude) | 已实现亏损交易的平均亏损率 | 用户可承受的单笔亏损范围 |
| 波动偏好 (vol_preference) | 买入价格的变异系数 | 偏好平稳标的 vs 高波动标的 |
| 趋势偏好 (trend_preference) | 均价相对首次出现价格的偏离度 | 追涨倾向 vs 抄底倾向 |

**关键性质**：不直接估计用户的风险承受能力，而是从交易模式中反推。

## 四、β 超参数：行为偏好 vs 资产偏好

用户侧重点可能不同：有些用户更在意"这个策略的操作风格是否像我"，有些更在意"这个策略投资的东西是否像我喜欢的"。

引入超参数 β ∈ [0, 1]：

```
匹配特征向量 = β × 行为特征 + (1-β) × (资产偏好 + 风险代理)
```

| β 值 | 用户画像 | 匹配侧重 |
|------|---------|---------|
| 0.0 | 信任产品推荐 | 完全匹配资产偏好 |
| 0.5 | 均衡（默认） | 行为与偏好并重 |
| 1.0 | 坚持自我风格 | 完全匹配行为特征 |

β 的确定方式：未来通过问卷调查 + 交易数据建模联合估计，当前取默认值 0.5。

## 五、相似度度量：径向惩罚余弦

### 5.1 为什么不用欧氏距离

欧氏距离对模长敏感：PCA 投影后，"极端程度"不同的用户离所有策略的距离都会变大。但极端程度本身不是"不匹配"的信号 — 一个行为画像与某策略高度一致但更极端的用户，应该被认为匹配度高。

### 5.2 为什么不用纯余弦相似度

纯余弦忽略模长信息。两个方向一致但极端程度悬殊的实体，会得到与模长接近时相同的相似度，这丢失了"风格极端程度差异"的信息。

### 5.3 径向惩罚余弦公式

$$\text{sim}(u, s) = \underbrace{\frac{u \cdot s}{\|u\| \cdot \|s\|}}_{\text{cosine（方向一致度）}} \times \underbrace{\exp\left(-\lambda \cdot \left|\log\frac{\|u\|}{\|s\|}\right|\right)}_{\text{radial penalty（模长惩罚）}}$$

**性质**：
- 方向一致 + 模长接近 → 满分
- 方向一致 + 模长悬殊 → 降权（惩罚因子 < 1）
- 方向相反 → 负值（保留"风格相反"的信息）
- λ = 0 时退化为纯余弦
- λ 默认取 1.0

**对称性**：用 log 比而非比值本身，保证了 ‖u‖=2‖s‖ 和 ‖s‖=2‖u‖ 时惩罚相同。

## 六、完整分析流程

```
┌──────────────────────┐
│  STEP 1: 数据加载      │  策略: daily_value.csv + trades.csv (7个)
│                       │  用户: 模拟账户A/B/C.xlsx (3个)
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  STEP 2: 策略特征提取  │  三层12维特征，直接从净值+交易数据计算
│                       │  行为(6) + 偏好(3) + 风险代理(3)
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  STEP 3: 用户特征提取  │  相同12维特征，仅从交易流水计算
│                       │  FIFO 配对 → 闭环交易 → 行为统计
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  STEP 4: β 加权 + PCA  │  β=0.5: 行为×0.5, 非行为×0.5
│                       │  标准化 → PCA → 保留90%方差的前k主成分
│                       │  投影: 去均值但不缩放（保留模长）
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  STEP 5: 相似度计算    │  径向惩罚余弦(λ=1.0) — 主度量
│                       │  余弦相似度 — 对照
│                       │  欧氏距离 — 对照
└──────────┬───────────┘
           ▼
┌──────────────────────┐
│  STEP 6: 推荐+可解释   │  每个用户 Top-3 策略
│                       │  维度级归因 + 弹窗话术
└──────────────────────┘
```

### 6.1 关键实现细节

**FIFO 配对**：对每个标的，按时间顺序将买入和卖出逐笔匹配。如果用户卖出某股票但之前没有买入记录（说明数据开始前已持有），则跳过 — 这避免了虚假做空和无法计算收益率的问题。

**PCA 投影保留模长**：PCA 在标准化数据上拟合以确定主成分方向，但投影时使用**去均值但不缩放**的数据。这样 PCA 空间的模长反映了实体在特征空间中偏离均值的"极端程度"，而非被标准化压缩到原点附近。

**ETF 识别**：由于用户交易记录中可能没有标的名称字段（仅 6 位代码），用代码前缀模式匹配识别 ETF（15/51/56/58/12 开头）。

### 8.4 结果合理性分析

**User A（长线低频，持仓 52 天，换手率 0.277/天）**：
- 推荐宽指配置：换手率 0.27/天（最低）、持仓 14 天（最长之一） — 风格高度一致
- 维度级归因：换手率差异仅 1.5%，买卖对称性差异 13.5%
- 径向惩罚余弦 vs 欧氏距离一致推荐宽指配置，纯余弦因 User A 模长较大推荐 HS300

**User B（超短线高频，持仓 2 天，换手率 3.674/天）**：
- 推荐 HX_朝花夕拾：唯一持仓仅 2 天的策略 — 操作频率完美匹配
- 维度级归因：持仓周期差异 0%、分仓均匀度高度一致
- 径向惩罚余弦和纯余弦一致，欧氏距离因模长影响推荐 GZ2000

**User C（短线中频，持仓 5 天，胜率 67% 最高）**：
- 推荐 HX_朝花夕拾：同样匹配其短线操作特征
- 维度级归因：买卖对称性差异最小、分仓均匀度高度一致
- 对 CYB300 和 HS300 的余弦均为负值，意味着风格方向相反

## 九、PCA 降维结果

| 主成分 | 解释方差比 | 累计方差 | 主导特征（载荷最高） |
|--------|-----------|---------|-------------------|
| PC1 | 42.4% | 42.4% | 持仓周期(+), 换手率(-), 处置效应(+) |
| PC2 | 27.7% | 70.1% | 亏损幅度(+), ETF 偏好(+), HHI(-) |
| PC3 | 14.8% | 84.9% | 趋势偏好(+), 买卖对称性(-), 胜率(-) |
| PC4 | 6.9% | 91.8% | 分仓均匀度(+), 价格偏好(+) |

**前 4 主成分解释 91.8% 的方差**，足以表征用户和策略的风格差异。

**PC1 的物理含义**：**"交易节奏"** — 持仓周期越长、换手率越低、处置效应越强（赚了不跑亏了扛）→ PC1 值越大。
**PC2 的物理含义**：**"资产类型"** — 亏损幅度大、ETF 占比高、持仓分散 → PC2 值越大。

## 十、局限性与改进方向

### 10.1 当前局限

1. **用户数据量有限**：仅 3 个模拟用户，且交易记录的时间跨度约 15 个月（2025.01-2026.04），不足以捕捉完整牛熊周期下的行为变化。

2. **无个股日频行情数据**：用户未卖出持仓的市值用买入价估值，存在系统性偏差。如果后续接入个股行情，可以精确计算每日持仓市值，重建更准确的 NAV 曲线。

3. **β 超参数未校准**：当前取默认值 0.5，未通过问卷或交易数据建模估计最优值。

4. **行业维度缺失**：当前用 ETF 占比代理行业偏好，缺少申万行业分类的精确映射。

5. **用户资金数据缺失**：当前无法得到用户初始资金，且数据量小，给用户既有策略收益率的估计不准确。后续可能可以采用有初始资金的数据，
训练机器学习模型，用于在未知用户初始资金的情况下估计（预测）用户初始资金。

### 10.2 改进方向

1. **接入问卷数据**：通过风险测评问卷获取用户的风险承受等级、收益期望、投资期限等信息，用于校准 β 超参数。

2. **滚动窗口匹配**：不使用全量历史交易，而是用最近 N 天的数据做匹配，观察匹配结果随时间的稳定性。

3. **行业维度匹配**：在获取个股行业分类后，增加 28 维行业分布向量，用 JS 散度计算行业相似度，与 PCA 空间结果做融合。

4. **Word2Vec 交叉验证**：团队正在进行的机器学习路线（交易序列 Token 化 + Word2Vec 嵌入）完成后，与本报告的统计方法做一致性分析。

5. **基于添加用户既有收益率后的策略推荐说辞**：在足够规模的带期初期末资金数的用户数据上训练期初资金数预测模型，随后基于用户现有收益率设计更有依据、更商业的推荐说辞。

## 十一、结论

1. **基于行为特征的偏好匹配方案可行**：即使在无法获取用户期初资金和绝对收益的情况下，仅凭交易流水中的行为特征（持仓周期、换手率、集中度、胜率等）已经能够有效刻画用户画像并与策略匹配。

2. **径向惩罚余弦优于纯欧氏/纯余弦**：同时捕捉方向一致度和模长差异，匹配结果既不受极端模长影响，也不丢失"极端程度"信息。

3. **匹配结果具有金融可解释性**：
   - User A（长线低频）→ 宽指配置（长线低频 ETF 策略）
   - User B（短线高频）→ 朝花夕拾（超短线个股策略）
   - User C（短线中频）→ 朝花夕拾 / GZ2000（中线中频策略）
   - 每个推荐都能在特征维度级归因，说明"为什么匹配"。

4. **"风格一致带来信任"的假设得到支持**：匹配度最高的策略在行为特征上与用户最接近，这意味着用户可以用自身的投资经验来理解推荐策略，降低认知摩擦。
