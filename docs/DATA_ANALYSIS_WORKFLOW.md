# 用户画像-投资策略匹配：完整数据分析工作流

> 基于三层特征体系 + PCA + 径向惩罚余弦相似度的策略推荐方案
> 日期：2026-05-24 | 版本：V3.0
>
> **架构**：三条并行路线 — 统计 PCA（主线）、LSTM 序列风格（辅线）、融合加权（推荐默认）
> **策略池**：44 种（7 原始净值 + 37 DLMethod Excel，通过适配器统一）
> **源数据**：统一存放于 `stats_data/` 目录

---

## 一、项目总览

### 1.1 核心思路

将**用户交易画像**和**投资策略画像**映射到同一个高维特征空间（12 维三层特征体系），
在该空间中用 **PCA 降维 + 径向惩罚余弦相似度** 衡量匹配度。

**三条并行路线：**

| 路线 | 方法 | 特征 | 匹配方式 |
|------|------|------|---------|
| **统计 PCA（主线）** | 12 维三层特征 + PCA 降维 | 行为 + 偏好 + 风险代理 | 径向惩罚余弦 |
| **LSTM 序列（辅线）** | BiLSTM 128 维嵌入 | 交易序列 token 风格 | LSTM Cosine |
| **融合加权（推荐默认）** | α=0.7 统计 + 0.3 LSTM | 两者结合 | Min-Max 归一化 + 加权 |

**关键流程：**
1. 从双源加载策略数据（目录格式 7 种 + Excel 格式 37 种）
2. 对策略和用户分别提取 **12 维三层特征**（仅从交易流水计算）
3. 对全部特征做 β 加权 + PCA 降维，取前 k 个主成分
4. 在 PCA 特征空间中用 **径向惩罚余弦** 计算匹配度
5. 为每个用户推荐 Top-N 最匹配的策略，附维度级归因

### 1.2 数据源清单

所有源数据统一存放在 `stats_data/` 目录下：

| 数据源 | 内容 | 规模 |
|--------|------|------|
| `stats_data/...products_export_20260518_163122/` | 7 个策略的 daily_value.csv + trades.csv + funds.csv | 每策略 ~812 天净值 + 数百笔交易 |

## 三、关键实现细节

### 3.1 三层特征体系（12 维）

#### 第一层：交易行为特征（6 维）

| 特征 | 计算方法 | 刻画含义 |
|------|---------|---------|
| 持仓周期 (holding_period) | FIFO 配对后，买入→卖出的天数**中位数** | 长线型 vs 短线型 |
| 换手率 (turnover_rate) | 总交易笔数 / 自然天数 | 活跃度 vs 静默型 |
| 买卖对称性 (buy_sell_ratio) | 买入金额 / 卖出金额 | 建仓型 vs 波段型 |
| 持仓集中度 (hhi_concentration) | 各标的交易金额占比的 **HHI 指数** | 集中 vs 分散 |
| 处置效应 (disposition_effect) | 已实现盈利单数占比 / 亏损单数占比 | 止损意识强弱 |
| 胜率 (positive_trade_ratio) | 闭环交易中盈利笔数 / 总闭环数 | 胜率偏好 |

#### 第二层：资产偏好特征（3 维）

| 特征 | 计算方法 | 刻画含义 |
|------|---------|---------|
| ETF 占比 (etf_ratio) | ETF 标的交易笔数占比 | 被动配置 vs 主动选股 |
| 价格区间偏好 (avg_price_preference) | 买入均价 | 低价股 vs 高价股偏好 |
| 分仓均匀度 (position_uniformity) | 1/(1+CV)，单笔金额变异系数的倒数 | 均匀分仓 vs 重仓集中 |

#### 第三层：风险代理特征（3 维）

| 特征 | 计算方法 | 刻画含义 |
|------|---------|---------|
| 亏损幅度 (avg_loss_magnitude) | 已实现亏损交易的平均亏损率 | 单笔亏损承受范围 |
| 波动偏好 (vol_preference) | 买入价格的变异系数（用户）/ NAV 日波动年化（策略） | 平稳 vs 高波动偏好 |
| 趋势偏好 (trend_preference) | 均价相对首次出现价格的偏离度（用户）/ NAV 趋势斜率（策略） | 追涨 vs 抄底 |

### 3.2 β 超参数加权

```python
# 行为特征 × beta，非行为特征 × (1 - beta)
匹配特征向量 = β × 行为特征 + (1-β) × (资产偏好 + 风险代理)
```

| β 值 | 侧重 |
|------|------|
| 0.0 | 完全匹配资产偏好 |
| 0.5 | 行为与偏好并重（默认） |
| 1.0 | 完全匹配行为特征 |

### 3.3 PCA 降维（保留模长）

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

# 1. 标准化确定主成分方向
scaler_std = StandardScaler()
X_scaled = scaler_std.fit_transform(X)
pca = PCA(n_components=0.90)  # 保留 90% 方差
pca.fit(X_scaled)

# 2. 去均值但不缩放的数据做投影（保留模长信息）
scaler_center = StandardScaler(with_std=False)
X_centered = scaler_center.fit_transform(X)
X_pca = X_centered @ pca.components_.T
```

**为什么保留模长**：PCA 空间的模长反映实体偏离均值的"极端程度"。
一个行为画像与某策略高度一致但更极端的用户，应该被认为匹配度高。
如果完全标准化，会丢失这种"极端程度"信息。

### 3.4 径向惩罚余弦相似度

```python
sim(u, s) = cos(u, s) × exp(-λ × |log(‖u‖ / ‖s‖)|)

# 余弦部分：方向一致度
cos_sim = (u · s) / (‖u‖ × ‖s‖)

# 径向惩罚：模长差异惩罚
penalty = exp(-λ × |log(‖u‖ / ‖s‖)|)
```

| 场景 | 余弦 | 惩罚因子 | 最终相似度 |
|------|------|---------|-----------|
| 方向一致 + 模长接近 | ~1.0 | ~1.0 | **满分** |
| 方向一致 + 模长悬殊 | ~1.0 | < 1.0 | **降权** |
| 方向相反 | 负值 | - | **负匹配** |

### 3.5 FIFO 买卖配对

```python
# 对每个标的，按时间顺序将买入和卖出逐笔匹配
# 如果用户卖出某股票但之前没有买入记录 → 期初底仓，跳过
# 这避免了虚假做空和无法计算收益率的问题
```

### 3.6 Excel 数据适配器（双源加载）

37 个 DLMethod 策略存储在 Excel 多 sheet 格式中，与原始 7 个策略的
目录 CSV 格式不兼容。`excel_strategy_loader.py` 适配器解决此问题：

1. **方向映射**：btype 列 '买入开仓'→BUY, '卖出平仓'→SELL
2. **数量处理**：SELL trades 数量为负 → `.abs()` 统一为正
3. **金额处理**：BUY 金额为负 → `.abs()` 统一为正
4. **净值构建**：从交易记录构建 NAV 序列（现金 + 持仓市值模型）
5. **Pipeline 兼容**：输出格式与原始 trades.csv / daily_value.csv 完全一致

### 3.7 三后端统一接口

```python
class MatchingBackend(ABC):
    def name(self) -> str: ...           # "statistical" / "lstm" / "fusion"
    def fit(self, strategy_features, strategy_nav=None): ...
    def predict(self, user_features, beta=0.5, top_n=3, industry_vector=None): ...
    def get_all_metrics(self, user_features, beta=0.5): ...
```

所有后端实现相同接口，支持运行时切换和交叉验证。

## 六、扩展方向

### 6.1 行业维度匹配
接入申万行业分类后，增加 28 维行业分布向量，用 JS 散度计算行业相似度，
与 PCA 空间结果做融合。

### 6.2 滚动窗口匹配
使用最近 N 天（30/60/120 天）的数据做匹配，观察匹配结果随时间的稳定性。
已通过 `ProfileService.update_profile_with_trades(window_days=N)` 实现。

### 6.3 冷启动处理
对于没有交易数据的新用户，使用问卷结果映射到特征空间。
三级问卷体系（L1 必填 5 题，L2/L3 可选）支持渐进式完善画像。

### 6.4 权重优化
初始权重基于金融常识设定。
如果有用户反馈数据（如订阅转化率），可以用贝叶斯优化调整权重。
