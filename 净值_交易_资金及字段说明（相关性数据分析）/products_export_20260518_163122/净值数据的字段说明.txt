2、 关于总资产
每个策略经过一段时间都会做一次‘除权’，因此在funds.xlsx里面存在running_delta字段，表示截止到该日累计除权提取现金的金额；
‘除权’的目的是防止某些策略因为长时间运行，导致总资产增加，而在交易中因为ticks数据撮合无法买入（或部分成交）的影响；
‘除权’时间、金额可以在净值图中找到，不在此罗列。

二、字段说明


## 1. 文档目的

本文档用于说明策略产品数据的目录结构、文件生成规则，以及各类 CSV 文件的用途和字段含义。

---

## 2. 数据目录规则

每个策略产品单独一个文件夹，文件夹名称为对应策略的 `product_id`,如：GZ2000_综合_v10

每个文件夹包含3个csv文件：funds.csv\ trades.csv\daily_value.csv





---

## 3. 当前支持的数据文件类型

当前支持三类 CSV 数据文件：

| 文件名            | 数据类型 | 主要用途                                         |
| ----------------- | -------- | ------------------------------------------------ |
| `funds.csv`       | 资金数据 | 查看每日资金余额、单日出入金、累计出入金         |
| `trades.csv`      | 交易明细 | 核对逐笔买卖成交记录                             |
| `daily_value.csv` | 策略净值 | 查看每日净值、总资产、现金、持仓市值和数据完整性 |

---

## 4. funds.csv 资金数据说明

### 4.1 文件用途

`funds.csv` 用于记录某个策略产品的资金流水和累计出入金结果，主要用于查看每日资金余额、单日出入金和累计出入金情况。

### 4.2 字段说明

| 字段名          | 字段含义     | 说明                                            |
| --------------- | ------------ | ----------------------------------------------- |
| `product_id`    | 策略 ID      | 当前策略产品的唯一标识，也是策略产品目录名。    |
| `product_name`  | 策略名称     | 策略产品的人类可读名称，便于识别。              |
| `fund_date`     | 资金日期     | 当前资金记录所属日期，格式通常为 `YYYY-MM-DD`。 |
| `cash`          | 当日现金余额 | 当前日期对应的账户现金值。                      |
| `delta`         | 当日出入金   | 当天新增或减少的资金变动值。                    |
| `running_delta` | 累计出入金   | 从起始记录累计到当前日期的 `delta` 总和。       |

---

## 5. trades.csv 交易明细说明

### 5.1 文件用途

`trades.csv` 用于记录某个策略产品的交易明细，主要用于核对逐笔买卖记录、成交价格、成交数量和成交时间。

### 5.2 字段说明

| 字段名         | 字段含义 | 说明                                                        |
| -------------- | -------- | ----------------------------------------------------------- |
| `product_id`   | 策略 ID  | 当前策略产品的唯一标识。                                    |
| `product_name` | 策略名称 | 策略产品的人类可读名称，便于识别。                          |
| `symbol`       | 证券代码 | 交易标的代码。                                              |
| `name`         | 证券名称 | 交易标的名称。                                              |
| `side`         | 买卖方向 | 通常为 `BUY` 或 `SELL`。                                    |
| `price`        | 成交价格 | 当前交易的成交单价。                                        |
| `qty`          | 成交数量 | 当前交易的成交数量。                                        |
| `amount`       | 成交金额 | 当前交易的总成交金额。                                      |
| `trade_time`   | 成交时间 | 交易发生的完整时间戳。                                      |
| `trade_date`   | 成交日期 | 从 `trade_time` 提取出的交易日期，格式通常为 `YYYY-MM-DD`。 |

---

## 6. daily_value.csv 策略净值说明

### 6.1 文件用途

`daily_value.csv` 用于记录某个策略产品的每日净值和总资产快照，主要用于查看净值曲线、资产规模、现金余额、持仓市值和数据完整性。

### 6.2 字段说明

| 字段名              | 字段含义       | 说明                                                         |
| ------------------- | -------------- | ------------------------------------------------------------ |
| `product_id`        | 策略 ID        | 当前策略产品的唯一标识。                                     |
| `product_name`      | 策略名称       | 策略产品的名称，便于识别。                                   |
| `date`              | 净值日期       | 当前净值快照所属日期，格式通常为 `YYYY-MM-DD`。              |
| `nav`               | 单位净值       | 系统计算后的策略单位净值。                                   |
| `total_value`       | 总资产         | 当日总资产，通常等于持仓市值加现金。                         |
| `market_value`      | 持仓市值       | 当日全部持仓的市值总和。                                     |
| `cash`              | 现金           | 当日现金余额。                                               |
| `running_delta`     | 累计出入金     | 截至当日的累计出入金。                                       |
| `initial_capital`   | 初始资金       | 策略初始本金，用于净值计算口径。                             |
| `data_completeness` | 数据完整性标记 | 标识当天净值基于哪些数据计算得出（详细见6.3data_completeness 取值说明）。 |

### 6.3 data_completeness 取值说明

| 取值             | 含义       | 说明                             |
| ---------------- | ---------- | -------------------------------- |
| `full`           | 数据完整   | 资金和持仓数据都齐全。           |
| `positions_only` | 仅有持仓   | 只有持仓数据，没有完整资金数据。 |
| `funds_only`     | 仅有资金   | 只有资金数据，没有完整持仓数据。 |
| `none`           | 无可用数据 | 当天没有可用于计算净值的数据。   |
